 **학사 관리 프로젝트**

- 요구사항 
학교 구성원 정보를 관리하는 시스템을 구축하고자 한다
학교 구성원은 학생 , 교사 , 직원이 있다 
학생은 전화번호 이름 주소 학번 정보로 구성된다 
교사는 전화번호 이름 주소 과목 정보로 구성된다 
직원은 전화번호 이름 주소 부서 정보로 구성된다 

-시스템 주요 기능
1. 구성원 등록(추가) 
학교 구성원(학생,교사,직원) 정보를 추가할 수 있다 
전화번호는 유일해야 하므로 등록(추가)시 기존 구성원 전화번호가
동일한 번호가 있을 경우 추가되어서는 안된다 
0101231234 tel 이 중복되므로 등록될 수 없습니다!  --> 메세지 출력 


2. 전체 구성원 리스트 출력 

3. 구성원 검색 
전화번호로 구성원 정보를 검색할 수 있다 
구성원 정보가 없을 때 null을 반환한다 

4. 구성원 삭제
전화번호로 구성원 정보를 삭제할 수 있다

일정 - 설계 class diagram 토의 후 작성 및 발표
설계 및 구현 소요시간

1. class diagram 도출
- VO(Value Object) 계열 클래스 구현 ( Person , Student , Teacher , Employee )

2. 테스트 클래스 ( TestSchoolService ) 와 비즈니스 로직 클래스 ( SchoolMemberAdministrationService )를 정의
3. 구성원 정보를 추가하고 전체 리스트 출력을 구현해서 확인한다.
	( 중복확인과 같은 제약조건은 일단 뺀다.)
	
4. 구성원 정보를 tel을 이용해 검색한다

5. 구성원 정보를 tel을 이용해 삭제한다.

3. 4. 5. 요구사항의 공통적인 기능은 구성원 정보를 저장하는 리스트에 중복되는 tel이 있는지 없는지 확인하고
있다면 리스트의 몇번째 요소인지를 확인하는 것이다.
이러한 공통기능을 수행하는 별도의 메서드를 만들고 3. 4. 5. 구현시 재사용하면 개발생산성이 높아질수있다.
	public int findIndexByTel(String tel){}
	위 메서드는 tel 매개변수에 전달된 전화번호와 리스트 요소의 전화번호가 일치하지 않으면 -1을 반환,
	일치하는 요소가 있으면 해당 인덱스를 반환한다.
	
	위 메서드를 구현하고 Test 클래스에서 잘 동작하는지 단위테스트를 한 후에 각 요구사항에 적용한다.
